Security Statement – Jira Issue Reminder

Last updated: 23 November 2025

1. Security Program

We maintain an information-security program aligned with ISO 27001 and SOC 2 Type II controls, covering governance, risk management, asset management, access control, cryptography, operations security, incident response, and business continuity.

2. Secure Development Lifecycle (SDLC)

  • All code is stored in a private Git repository with branch-protection and mandatory pull-request reviews.
  • We run automated SAST/DAST scans (GitHub Advanced Security + OWASP ZAP) on every build.
  • Dependencies are scanned daily; high-risk vulnerabilities are patched within 14 days.
  • Production releases use Atlassian Forge’s built-in CI/CD pipeline; no customer artifacts are deployed to third-party servers.

3. Data Handling & Isolation

  • The app operates entirely inside Atlassian Forge; tenant data never leaves the Atlassian cloud.
  • We retain only the JQL queries and minimal user identifiers required for reminder functionality.
  • Data is encrypted in transit (TLS 1.3) and at rest (AES-256) by Forge storage.
  • Multi-tenant isolation is enforced by Forge’s runtime sandbox; no cross-tenant data access is possible.

4. Access Control

  • The production environment is managed by Atlassian; we have zero shell or infrastructure access.
  • Administrative access to the Forge developer console is protected by SSO + TOTP.
  • Role-based permissions are reviewed quarterly.

5. Logging & Monitoring

  • Forge automatically emits audit logs for all install, uninstall, and configuration events.
  • Security logs are retained for 180 days and forwarded to Atlassian’s centralized pipeline.
  • Anomalous-activity alerts are configured via Forge observability hooks.

6. Incident Response

  • Security incidents are reported to Atlassian within 2 hours at dev-ecosystem-security@atlassian.com.
  • Customers are notified within 24 hours if their data is impacted.
  • Post-mortem and root-cause analysis are completed within 5 business days.

7. Penetration Testing & Bug Bounty

  • Annual external penetration tests are performed by a CREST-accredited firm.
  • Critical and high findings are remediated before re-testing.
  • Researchers may report vulnerabilities through our VDP (see section 9).

8. Compliance & Certifications

  • Hosted on Atlassian Forge, which is ISO 27001, SOC 2 Type II, and GDPR compliant.
  • We sign Atlassian’s Data Processing Addendum and leverage Atlassian’s sub-processor list.
  • No additional sub-processors are used by the app.

9. Reporting Security Issues

If you discover a vulnerability, please email wbohome@outlook.com with:

  • Summary and impact
  • Steps to reproduce
  • Possible remediation

We follow coordinated disclosure and will credit researchers unless anonymity is requested.

10. Contact Us

For any security or compliance questions:
Email: wbohome@outlook.com
Response time: ≤ 24 hours on business days.
